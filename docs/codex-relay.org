This file is the execution authority for the first Codex Relay release. Read
=docs/2026-07-14-codex-relay-design.md= and =docs/codex-app-server.txt= before
changing code. Work through L1 and L2 headings in order. Do not invent alternate
frameworks, persistence mechanisms, endpoints, names, or scope.

- Goal :: Build one mobile-first TypeScript application that supervises Codex
  app-server processes, restores their threads, provides reliable chat over a
  relay WebSocket, and exposes safe Git status/fetch/push operations.
- Architecture :: TypeScript modular monolith. Vertical slices own REPR HTTP
  endpoints and use cases. Domain rules remain framework-free. All filesystem,
  process, Codex, SQLite, Git, and clock access goes through ports/adapters.
- Runtime stack :: Node.js 24+, Svelte 5, Vite 8, Fastify 5, =ws= 8, Zod 4,
  built-in =node:sqlite=, Vitest 4, and Playwright 1.61.
- Required workflow :: Before implementation, create branch =codex-relay= from
  the current commit and run the existing tests. For each L2: set it to WIP,
  write the failing test, run it and observe the intended failure, implement the
  smallest code, run touched tests, commit with the prescribed conventional
  commit, then set the L2 to DONE. After all L2 under an L1 are DONE, run the
  full suite before marking the L1 DONE. Never commit this plan or other files
  under =.gestalt=.
- Stop condition :: If a generated Codex type or current CLI behavior conflicts
  with this plan, stop and ask for help. Do not cast through =any=, silently
  ignore a known protocol message, weaken a test, or replace an invariant.

/Global constraints/

- Serve one plain-HTTP origin. Do not add TLS, CORS, accounts, or application
  authentication.
- Bind to =0.0.0.0:8787= by default. Support =--host=, =--port=, =--cwd=, and
  =--data-dir= overrides.
- Treat only canonical immediate children of the Relay Root as workspaces.
  Reject the root, nested paths, and symlinks resolving outside the root.
- Discover profiles with =codex-profile status --json=. Do not manage profiles.
- Launch one =codex-profile cli <profile> app-server --stdio= child per active
  Relay Session. Never expose app-server directly to the browser.
- Use REST for mutations and one relay WebSocket for server-originated events.
- Preserve session/thread metadata in SQLite and Codex history in Codex homes.
- Support command/file/permission approvals and =requestUserInput=.
- Throttle successful Git fetches for 60 seconds. Never force-push and never
  create an upstream.
- Use native doc-comments on exported functions. Comments explain purpose and
  invariants, not syntax.
- Prefer platform APIs. Do not add an ORM, query builder, state library, client
  router, CSS framework, CORS plugin, authentication package, or Markdown/HTML
  renderer.
- Never use =exec= or =shell: true= for child processes. Use =spawn= or
  =execFile= with argument arrays.
- Operational logs must not contain prompts, generated content, approval
  answers, command output, environment values, or Git credentials.
